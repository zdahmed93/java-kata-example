## Top 3 Prioritized Refactorings

1. Wire up the DTO layer and fix entity leakage (Correctness + Security)
The SubscriptionResponse DTO exists but is never used. The controller returns raw JPA entities, which exposes internal model details, risks lazy-load exceptions during serialization, and breaks the API contract. Implement a mapper (MapStruct is already in pom.xml) and make every endpoint return a proper DTO. This is the single most impactful structural fix.

2. Replace @Autowired field injection with constructor injection and add @Valid + a @RestControllerAdvice (Robustness)
Field injection makes the code untestable without a Spring context. Constructor injection with final fields is the Spring Boot standard. Pair this with adding spring-boot-starter-validation, annotating request bodies with @Valid, and adding a @RestControllerAdvice that maps exceptions to RFC 7807 ProblemDetail responses. Without this, any bad input produces an unhandled 500.

3. Implement the test suite for the proration logic and re-enable Kafka (Completeness)
The kata explicitly requires demonstrating correctness. The proration calculation (the core business logic) has a latent ArithmeticException bug and zero test coverage. Write unit tests for SubscriptionService covering: normal proration, upgrade vs downgrade, already-expired cycle edge case, and non-ACTIVE guard. Simultaneously, uncomment and fix the Kafka publisher — it is a stated hard requirement that is currently dead code.
