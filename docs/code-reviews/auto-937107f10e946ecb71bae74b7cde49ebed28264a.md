## Top 3 Prioritized Refactorings

1. Fix the entity leak and wire the mapper (Architecture — highest impact)
The controller returns raw JPA entities. This is the most critical architectural flaw. Add a @Mapper (MapStruct) interface, map Subscription → SubscriptionResponse in the service, and return the DTO from the controller. This also eliminates the LazyInitializationException risk during serialization.

2. Write tests for the proration business logic (Correctness — kata requirement)
The README explicitly requires demonstrating the solution works. The proration calculation is the heart of the kata. A plain JUnit 5 unit test with no Spring context needed — just instantiate the service with mocked DAOs and assert the nextCycleWithProrated value for upgrade and downgrade scenarios. Also fix the missing RoundingMode in divide() before writing those tests or they will fail on edge cases.

3. Replace @Autowired field injection with constructor injection everywhere + uncomment Kafka (Testability & Completeness)
Every class uses field injection, making unit testing impossible without a full Spring context. Switching to constructor injection is a one-line change per class and unlocks proper unit testing. Simultaneously, the Kafka publisher must be uncommented and wired — it is a hard requirement of the kata and is currently completely disabled.
