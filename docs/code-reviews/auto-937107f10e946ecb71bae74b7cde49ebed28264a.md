Kafka is a hard requirement of the kata and is entirely commented out. SubscriptionEventPublisher must be re-enabled as a @Component, injected via constructor, and called after every subscription creation and change. Both endpoints return raw Subscription entities. This causes lazy-load serialization issues and leaks the persistence model. The existing SubscriptionResponse DTO must be fixed (remove entity fields, use primitives/value types), a mapper must be written (MapStruct is already declared), and the controller must return SubscriptionResponse. This is the single most impactful structural fix.

3. Add a global exception handler and input validation (Critical)
Add spring-boot-starter-validation to pom.xml, annotate request record fields with @NotNull/@Positive, add @Valid in the controller, and create a @RestControllerAdvice that maps EntityNotFoundException → 404, IllegalStateException → 409, and MethodArgumentNotValidException → 400, all using Spring 6's native ProblemDetail (RFC 7807).
