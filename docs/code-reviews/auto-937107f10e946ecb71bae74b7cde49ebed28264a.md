## Top 3 Prioritized Refactorings

1. Fix the entity leak and wire the DTO layer (Blocker)
The controller returns raw JPA entities. SubscriptionResponse exists but is unused. The service interface must return DTOs, not entities. This is the single most impactful change: it fixes the API contract, prevents serialization issues with lazy associations (LazyInitializationException or N+1), and decouples the persistence model from the API model. Implement the MapStruct mapper that's already declared as a dependency.

2. Add a @RestControllerAdvice and input validation (Blocker)
There is zero error handling. Any bad input (null IDs, non-existent customer, changing an already-cancelled subscription) produces an unstructured 500. Add spring-boot-starter-validation, annotate DTOs with @NotNull, add @Valid in the controller, and create a GlobalExceptionHandler returning ProblemDetail (RFC 7807, natively supported since Spring 6). This is table-stakes for any production API.

3. Write tests for the proration business logic (Critical)
The proration calculation is the core deliverable of this kata and it has zero test coverage. It also contains a latent ArithmeticException bug (divide without RoundingMode). A parameterized @ParameterizedTest covering upgrade, downgrade, mid-cycle, and end-of-cycle scenarios would both validate correctness and demonstrate understanding of the business rule. This is what the interviewers will ask about first.
