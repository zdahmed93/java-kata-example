## Top 3 Prioritized Refactorings

1. Fix the entity leak and wire the DTO layer (highest impact)
The controller returns raw JPA entities. This causes LazyInitializationException or silent N+1 queries at serialization time, and exposes the persistence model to API consumers. The SubscriptionResponse record already exists — implement a mapper (MapStruct is already in pom.xml) and use it. This is the single most impactful change for correctness and production safety.

2. Replace @Autowired field injection with constructor injection + add @RestControllerAdvice
Field injection makes the service untestable without a Spring context. Switching to constructor injection (one @RequiredArgsConstructor from Lombok) unblocks unit testing. Pair this with a @RestControllerAdvice that maps EntityNotFoundException → 404 and IllegalStateException → 409, so the API returns structured errors instead of 500 stack traces.

3. Write tests for the proration business logic and uncomment Kafka
The proration calculation in changeSubscription is the core business requirement of the kata and has zero test coverage. A plain JUnit 5 unit test with Mockito stubs for the DAOs would cover this in ~30 lines. Simultaneously, the Kafka publisher must be uncommented and wired — it is the only technical requirement explicitly stated in the kata and is currently dead code.
