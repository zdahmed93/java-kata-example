## Top 3 Prioritized Refactorings

1. Fix the build — the project likely does not compile

spring-boot-starter-webmvc, spring-boot-starter-webmvc-test, and spring-boot-starter-data-jpa-test are not valid Maven artifact IDs. Replace them with spring-boot-starter-web and spring-boot-starter-test. Also delete SubscriptionStatus2.java which creates a duplicate JPA entity mapping for subscription_plan. Nothing else matters if the project doesn't build.

2. Stop leaking JPA entities through the API and wire Kafka

The controller returns raw Subscription entities. Jackson will attempt to serialize lazy-loaded associations, causing either LazyInitializationException or infinite recursion. Map to SubscriptionResponse (as a proper record with no entity references) in the service layer. At the same time, uncomment and fix the SubscriptionEventPublisher — Kafka is a stated mandatory requirement and is currently fully disabled.

3. Add a @RestControllerAdvice and input validation

Without these, any bad input (null IDs, missing fields) produces an uncontrolled 500 with a stack trace. Add spring-boot-starter-validation, annotate request records with @NotNull, add @Valid on controller parameters, and add a @RestControllerAdvice that maps EntityNotFoundException → 404, IllegalStateException → 409, and validation errors → 400 with structured ProblemDetail bodies.
